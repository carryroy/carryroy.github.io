GitHub的使用

2020-08-08 3 min read

GitHub

命令 作用
git status 查看状态
git init 初始化git仓库
git add a.md 提交a.md到仓库
git commit -m 'first commit' 正式提交
git log 日志
git log --pretty=oneline 简洁版日志
git branch 查看分支
git branch a 新建分支a
git checkout a 切换到分支a
git checkout -b a 新建一个a分支，并且自动切换到a分支
git merge a 把a分支的代码合并过来
git branch -d a 把a分支删除
git branch -D 强制删除a分支
git tag v1.0 在当前代码状态下新建了一个v1.0的标签
git tag 查看历史tag记录
git checkout v1.0 切换到v1.0 tag的代码状态
git checkout a.md 恢复删除的a.md
git remote -v 查看当前项目有哪些远程仓库
git diff a.md 查看修改了什么
git reflog 查看commit等操作的版本号
git reset -–hard 版本号 回退到某个版本，如最新版本
git reset -–hard HEAD^ 回退到上个版本，^^上上个版本
git reset -–hard HEAD~100 回退到前100个版本

1.ssh-keygen -t rsa
指定 rsa 算法生成密钥，接着连续三个回车键（不需要输入密码），然后就会生成两个文件 id_rsa 和 id_rsa.pub ，而 id_rsa 是密钥，id_rsa.pub 就是公钥
接下来要做的是把 id_rsa.pub 的内容添加到 GitHub 上，这样你本地的 id_rsa 密钥跟 GitHub 上的 id_rsa.pub 公钥进行配对，授权成功才可以提交代码

2.git push origin master
第一次推送master分支时，加上了 –u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令

3.git push origin master
把本地代码推到远程 master 分支

4.git pull origin master
意思就是把远程最新的代码更新到本地，一般我们在 push 之前都会先 pull ，这样不容易冲突

5.commit前设置名字邮箱
git config —global user.name "stormzhang"
git config —global user.email "stormzhang.dev@gmail.com"

6.克隆自己的项目
git clone git@github.com:stormzhang/test.git

7.关联本地已有项目
git remote add origin git@github.com:stormzhang/test.git
